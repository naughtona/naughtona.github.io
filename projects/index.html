Expendibots Game Engine

I developed two game-playing agents, as well as functionality for a human user, that play a novel 8x8 board game, called Expendibots.

Notably, the AI game-playing agent is an exploration of the mtd-f search algorithm. It has been implemented within an iterative deepening time-based cut-off scaffold. It features several other optimisations, including a transposition table and an opening book.

Tools: (OOP) Python and Linux

View and test out the project here.

Note that the linked repository also includes a detailed write-up that explains my motivations for applying said algorithms to my game-playing agent within the context of Expendibots.


New York City Yellow Taxi Data -- Info Retrieval

I created a dictionary for New York City Taxi Data, where the underlying data structure is a Binary Search Tree (BST).

Amongst other things, such as an appreciation for time complexity analysis, this project aims to illustrate the efficiency of look-up operations when the search key is indeed the same as the key on which the BST is sorted.

Equally, it attempts to warn of the futility of implementing a BST when the search key differs from the BST key.

Tools: C, Makefiles, and Linux

View and test out the project here.


NIM Game Engine

I built a game engine that facilitates game play of two variants of NIM.

The system is designed to allow users to create AI players. Throughout the game, these players will choose their actions independent of the user and play a dominant strategy, where one exists.

Alternatively, you can create Human player, who plays according to your instructions via standard input.

Tools: Java and Linux
